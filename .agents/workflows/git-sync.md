// turbo-all

Этот воркфлоу автоматически собирает все изменённые файлы, пересобирает production-сборку для GitHub Pages, делает коммит и пушит в репозиторий.

**ВАЖНО:** `concept_v2/dist/` находится в `.gitignore`, поэтому нужно принудительно добавлять его через `git add -f`.
GitHub Pages обслуживает именно `dist/` — без пересборки пользователи видят старую версию.

Шаги, которые я выполню без запроса разрешений:
1. `git status` (чтобы понять, что изменилось)
2. `cd concept_v2 && npm run build` (пересобрать production dist для GitHub Pages)
3. `git add -f concept_v2/dist` (принудительно добавить dist, он в .gitignore)
4. `git add .` (добавить все остальные файлы)
5. `git commit -m "..."` (сделать коммит с описанием изменений)
6. `git push` (отправить на сервер)

GitHub Pages обновляется в течение 1–2 минут после пуша. Если видна старая версия — сделать hard refresh (Cmd+Shift+R).
