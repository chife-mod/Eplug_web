# Eplug Web — Architecture Overview

> **Назначение файла:** Глобальная карта логики и структуры проекта. Обновляется при добавлении новых функциональных/логических блоков.

---

## Стек технологий

| Слой | Технология |
|---|---|
| Разметка | Vanilla HTML5 |
| Стили | Vanilla CSS (без фреймворков) |
| Логика | Vanilla JavaScript (ES6+) |
| Сервер разработки | Python HTTP / Live Server |
| Хостинг / CDN | GitHub Pages |

---

## Структура проекта

```
Eplug_web/
├── docs/                   ← Документация (этот раздел)
│   ├── architecture/       ← Логика, потоки данных, архитектурные решения
│   └── components/         ← UI-компоненты, страницы, крупные сборки
├── UX/                     ← UX-хаб: точка входа, sitemap, wireframes
│   ├── index.html
│   ├── sitemap.html
│   └── wireframes/
├── assets/                 ← Медиа: логотипы, иконки, видео, изображения
├── materials/              ← Исходники: брифы, тексты, референсы
├── concept_v2/             ← Интерактивный прототип (Next.js / Vite)
├── scripts/                ← Вспомогательные скрипты
├── shared-header.js        ← Общий навигационный блок (переиспользуемый)
├── .agents/workflows/      ← AI-агент воркфлоу
├── PLAN.md                 ← Текущие задачи и план разработки
└── README.md               ← Пульт управления проектом
```

---

## Функциональные блоки

> Каждый блок ниже — отдельный файл в `docs/architecture/`.

| Блок | Файл | Описание |
|---|---|---|
| Shared Header | `shared-header.md` | Общий навигационный компонент, подключаемый через JS |
| Роутинг / Навигация | `routing.md` | Логика переключения между страницами |
| Данные / Контент | `data-flow.md` | Откуда берётся контент, как он рендерится |

---

## Соглашения

- **Документация идёт первой** — перед кодом всегда создаётся doc-файл.
- **Архитектура** описывает *как это работает* (логика, потоки данных, паттерны).
- **Компоненты** описывают *как это выглядит и используется* (UI, API, параметры).
- **Коммит** включает и код, и документацию вместе.
